# OctoAcme Role Communication Matrix Template

| Role                      | Communicates With                | Topics/Artifacts                   | Frequency          |
|---------------------------|----------------------------------|------------------------------------|--------------------|
| Project Manager           | Product Manager, Devs, Stakeholders | Status updates, risk register      | Weekly, ad-hoc     |
| Product Manager           | Project Manager, QA, Devs        | Backlog, requirements, priorities   | Weekly, sprint-wise|
| Developers                | Project Manager, Tech Lead, QA   | Design docs, testing, PRs           | Daily              |
| QA                        | Devs, Product Manager            | Test cases, acceptance tests        | Sprint, release    |
| Technical Lead            | Devs, PM, Product Manager        | Architecture, technical risks       | As needed          |
| Scrum Master              | Dev Team, PM                     | Standups, retros, blockers          | Daily/Sprint       |
| UX Designer               | Product Manager, Devs            | Prototypes, usability feedback      | Sprint, ad-hoc     |
| Business Analyst          | Stakeholders, Devs, PM           | Requirements, user stories          | Planning, as needed|
| Support/Customer Success  | Product Manager, QA, PM          | User pain, incidents                | Release, incident  |

_For each new project, fill out this matrix to clarify role interactions._
