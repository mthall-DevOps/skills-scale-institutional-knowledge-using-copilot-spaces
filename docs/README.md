# OctoAcme Project Management Processes

This repository centralizes OctoAcme's program and project management workflows, roles, communications strategies, and quality assurance practices. The goal is to convert tacit team knowledge into versioned, accessible artifacts that empower all team members and support consistent, repeatable project execution.

## Overview of Project Management at OctoAcme

OctoAcme leverages a lifecycle-based project management approach divided into five phases: Initiation, Planning, Execution, Release, and Close & Retrospective. Initiation begins with a lightweight Project One-pager to capture problem statements, key stakeholders, and success metrics, ensuring clear alignment before projects move forward. Planning includes scoping work into shippable increments, estimating with story points or T-shirt sizes, and populating a prioritized backlog that is used to guide implementation.

Day-to-day execution is managed using team rituals such as daily standups, weekly delivery syncs, end-of-sprint demos, and structured Kanban-style project boards (Backlog, Ready, In Progress, In Review, QA, Done). All work is tracked by small, peer-reviewed pull requests linked to issues and validated with automated tests, security scanning, and manual QA. Velocity tracking, burndown charts, and clear acceptance criteria help ensure timely delivery and quick problem escalation.

Key roles are clearly defined: Project Managers coordinate delivery and communication, Product Managers own the backlog and product vision, Developers build and test features, and QA validates releases. Communication is structured through regular syncs, stakeholder updates, and clearly defined escalation paths for blockers and risks. A continuous improvement culture is embedded by running retrospectives at each milestone, capturing learnings as actionable backlog items, and tracking progress on improvements in weekly syncs.

By centralizing these processes, OctoAcme accelerates onboarding, reduces single-person dependency risk, and provides all team members with equal access to structured, living process documentation.

## Purpose of this Repository

- Make all project management processes findable and consistent
- Convert tribal knowledge into standardized, versioned documentation
- Enable repeatable success across new initiatives and teams
- Connect real project decisions and rationale for future reference
- Lower onboarding friction and close process gaps

Explore the `/docs` folder for detailed guidance on each process phase, roles, communication templates, and checklists.
